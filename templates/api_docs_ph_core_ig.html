PH-Core Structure Definitions

GET /StructureDefinition/{profile_id}

Access PH-Core StructureDefinition resources directly via their canonical URLs. These define the profiles and constraints for Philippines Healthcare resources.

Note: These endpoints serve the Implementation Guide resources directly and support FHIR canonical URL resolution.

Available Structure Definitions

PH-Core Patient

ph-core-patient

Patient profile with indigenous people extension and PSGC address support

Indigenous People Extension

indigenous-people

Extension to indicate if a person belongs to an indigenous group

City/Municipality Extension

city-municipality

PSGC-compliant city/municipality address extension

Province Extension

province

PSGC-compliant province address extension

Example: Get PH-Core Patient Profile

curl -X GET "https://wah4pc-validation.echosphere.cfd/StructureDefinition/ph-core-patient"

Implementation Guide Information

About PH-Core IG

The Philippines Healthcare Core (PH-Core) Implementation Guide defines the minimum conformance requirements for FHIR resources used in Philippine healthcare systems. It includes profiles, extensions, and terminologies specific to the Philippine healthcare context.

Key Features

  • Indigenous people identification
  • PSGC-compliant address extensions
  • Philippines-specific terminologies
  • Healthcare facility naming systems

Compliance Requirements

  • Must include required PH-Core profiles in meta.profile
  • Address extensions must use PSGC codes
  • Indigenous people extension is required for Patient
  • Must validate against both FHIR R4 and PH-Core constraints
